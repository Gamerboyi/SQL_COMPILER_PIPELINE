How It Works

🔵 Stage 1 — Lexical Analysis

The lexer scans the raw SQL text character by character and groups them into tokens: keywords (SELECT, FROM), identifiers (table/column names), operators (=, >), literals (numbers, strings), and punctuation.

🟣 Stage 2 — Syntax Analysis (Parsing)

The parser reads the token stream and builds an Abstract Syntax Tree (AST) using recursive descent parsing. The AST represents the hierarchical structure of the SQL statement — which clauses contain which expressions.

🟢 Stage 3 — Semantic Analysis

The semantic analyzer validates the AST against the database schema. It checks that tables and columns exist, verifies type compatibility in comparisons, and ensures aggregate functions are used correctly with GROUP BY.

🟠 Stage 4 — Intermediate Code Generation

The ICG converts the AST into relational algebra — a mathematical notation used internally by databases. Operations like σ (selection), π (projection), and ⋈ (join) describe how data flows through the query plan.

🩷 Stage 5 — Query Optimization

The optimizer applies rules to improve the query plan: pushing filters closer to data sources (predicate pushdown), reordering joins by table size, folding constant expressions, and eliminating redundant operations.
